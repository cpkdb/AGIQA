## Grok Search

### 激活与路由

- 默认优先使用原生的 Web 工具作为完成快速检索与初步核验
- 当触发以下任意条件时，必须使用Grok Search：
  - 需要深度研究/横向对比/长文抽取
  - 原生 web 结果冲突或信息不足
  - 需要特定平台定向搜索(GitHub/Reddit/X 等)
  - 高风险问题需要多源交叉验证
- 最终关键结论必须回到官方源复核！

### 模型选择（强制）
**执行时机**：每次调用 `web_search` 或 `web_fetch` 前
**必须执行**：先调用 `switch_model` 切换到合适模型，默认使用`grok-4.1-thinking`

| 任务类型 | 模型                 | 触发词               |
| -------- | -------------------- | -------------------- |
| 快速查询 | `grok-4.1-fast`        | 简单问题、单一事实   |
| 深度研究 | `grok-4.1-thinking`  | 分析、对比、复杂问题 |
| 多方验证 | `grok-4.1-expert` | 需要多源交叉验证     |

**默认行为**：无明确指示时使用 `grok-4.1-thinking`

### 执行策略

**查询构建**：广度用 `web_search`，深度用 `web_fetch`，特定平台设 `platform` 参数
**搜索执行**：优先摘要 → 关键 URL 补充完整内容 → 结果不足调整查询重试（禁止放弃）
**结果整合**：交叉验证 + **强制标注来源** `[标题](URL)` + 时间敏感信息注明日期

### 错误恢复

连接失败 → `get_config_info` 检查 | 无结果 → 放宽查询条件 | 超时 → 搜索替代源

### 核心约束

- 强制 GrokSearch 工具 + 输出必含来源引用 + 失败必重试 + 关键信息必验证
- 禁止无来源输出 + 禁止单次放弃 + 禁止未验证假设
